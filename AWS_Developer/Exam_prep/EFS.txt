EFS: (Elastic Filye System) 

	Skálázható file storage service, amit felcsatolhatunk EC2 instanceok és containerek, ECS taskok alá.

	 With Amazon EFS, storage capacity is elastic, growing and shrinking automatically as you add and remove files. 
	 Your applications can have the storage they need, when they need it. Amazon EFS volumes are supported for tasks hosted on Fargate or Amazon EC2 instances.



	Az EBS-sel ellentétben ez az instance nem kötött Availability Zone-hoz, az EC2 instancok, amiket kiszolgál több AZ-ban is lehetnek egyszerre.	
	Nagyon drága (3x)
	CSAK Linux EC2 instance-okra csatolható fel.
	De csak a használatért fizetünk (pay-per use)
	Különböző módokban tud üzemelni:
		Performace:
			-General purpose(default): latency-sensitive use cases(web server, CMS, ..)
			-MAX I/O-higher latency,throughtput, highly parallel(big data, media processing)
		Throughtput:
			-Bursting: throughput scales with file system size
			-Provisioned: throughput fixed with a specified amount
	Storage Tiers:
		Standard: hétköznapi, azaz folyamatos, de kitett használatra is ez javasolt.
		Infrequent access: ha ritkán és teljesen random pillanatokban akarunk írni/olvasni.
